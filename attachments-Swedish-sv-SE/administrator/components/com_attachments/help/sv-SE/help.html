Introduktion

Med tillägget 'Attachments' för Joomla! kan du ladda upp filer som bilagor till artiklar. 'Attachments' består av ett insticksprogram för att visa bilagorna och en komponent för att ladda upp och underhålla bilagor. Det går att kontrollera vilka som kan se bilagorna eller ladda upp bilagor, liksom många andra inställningar för att öka tilläggets flexibilitet och användbarhet. Observera: alla inställningar kontrolleras i komponenthanteraren. Detta tillägg fungerar enbart i Joomla! 1.5 och senare.

Restriktioner för uppladdning

Alla filtyper kan inte laddas upp. Tillägget 'Attachments' kommer att stoppa uppladdning av filer som inte tillåts av Joomla!s Mediahanterare. För att se (eller ändra) vilka filtyper som tillåts, gå till Globala inställningar och klicka på System. I gruppen Mediainställningar kan du välja vilka filtyper och mime-typer som är tillåtna för uppladdning. 'Attachments' respekterar dessa inställningar men ignorerar Tillåtna bildtyper (filändelser). Attachments Inställningar

Alla inställningar för 'Attachments' sköts i komponenthanteraren. Gå in in adminstratörsgränssnittet och välj 'Attachments' under menyn Komponenter för att ändra inställningarna. Klicka på "Inställningar" till höger bland verktygen för att se listan på parameterar för tillägget. Det finns de här parametrarna:
  • Vem som kan se bilagor: Denna inställning bestämmer vilka som kan se länkarna till de bifogade filerna. För närvarande finns två val:
    1. 'Alla' - Om detta är valt kan alla besökare se länkarna till de bifogade filerna, oavsett om de är inloggade eller inte.
    2. 'Alla inloggade användare' - Om detta är valt kan länkarna bara ses av inloggade användare.
  • Vilka kan lägga till bilagor: Denna inställning kontrollerar vilka som kan lägga till bilagor till artiklar. Det finns två val::
    1. 'Endast artikelns författare' - Länken för att ladda upp bilagor är bara synlig för artikelns författare (och användare med högre behörighet som Redaktörer och Administratörer).
    2. 'Alla inloggade användare' - Länken för att ladda upp bilagor är synlig för alla besökare.
  • Publicera bilagorna omedelbart: Denna inställning för omedelbar publicering kontrollerar om nytillagda bilagor publiceras genast eller inte. Vid Ja blir bilagorna genast publicerade och synliga för alla besökare. Vid Nej publiceras inte bilagorna automatiskt. De måste då först publiceras av en Administratör via Adminstratörsgränssnittet för att bli synliga för besökare.
  • Varning för automatisk publicering: Om funktionen för automatisk publicering är avslagen (föregående funktion) kan det vara lämpligt att informera dem som laddar upp bilagor om detta, och hur de ska göra för att få bilagan publicerad. Skriv ett lämpligt meddelande här. Om fältet lämnas tomt levereras ett allmänt systemmeddelande att kontakta webbplatsens administratör för att få den nya bilagan publicerad.
  • Visa rubriker: Vid Ja visas beskrivande rubriker över kolumnerna på listan med bilagor.
  • Visa beskrivningar: Denna inställning kontrollerar om bilagans beskrivning ska visas i listan.
  • Visa vem som laddat upp bilagan: Visar användarnamnet på den som har laddat upp bilagan.
  • Visa filstorlek: Denna inställning kontrollerar om filstorleken kommer att visas i listan med bilagor.
  • Visa antal nedladdingar: Denna inställning kontrollerar om antal nedladdningar kommer att visas i listan med bilagor. VARNING: Detta val fungerar bara i säkert läge. I läge 'bilaga' anropas filerna direkt utan att passera Joomla!-koden. Därför är det inte möjligt att fånga och räkna antal nedladdningar i det läget. Av det skälet visas antal nedladdningar bara i läge 'Inline' när Ja är valt här.
  • Visa ändringsdatum: Vid Ja visas senaste ändringsdatum för filen i listan över bilagor till en artikel. Vid Nej visas inget datum.
  • Format för ändringsdatum: Du kan välja datumformat enligt PHP-funktionen strftime(). Sök på nätet med 'PHP strftime' för att hitta exempel. Det förvalda formatet (%x %H:%M) ger datum med 24-timmarsklocka så här; 4/28/2008 14:21. Slopa delen "%H:%M" för att ta bort klocktiden. Notera att PHP under MS Windows eller Linux kan stödja olika koder för tidsformat.
  • Namn för användardefinierade fält 1-3: Programmet ger möjlighet att defiera upp till tre användardefinierade fält för att lägga till ytterligare information om dina bilagor. Skriv in den önskade benämningen för vart och ett av fälten här. Radera namnet för att ta bort visning och redigering av fältet. Fälten kommer att visas i den ordning som de visas här. Fältnamnet får vara högst 40 tecken och fälten kan innehålla upp till 100 tecken.
  • Bilagornas listordning: Här väljer du sorteringsordning för bilagorna i listan. De flesta benämningarna är självbeskrivande:
    1. 'Filnamn' - Sorterar alfabetiskt på filnamn.
    2. 'Filstorlek (minst först)'
    3. 'Filstorlek (störst först)'
    4. 'Beskrivning'
    5. 'Skapad datum (äldst först)'
    6. 'Skapad datum (nyast först)'
    7. 'Ändrad datum (äldst först)'
    8. 'Ändrad datum (nyast först)'
    9. 'Bilage-ID' - Sorterar bilagorna efter deras ID i databasen. Effekten blir att bilagorna redovisas i den ordning de skapades.
    10. 'Användardefinierat fält 1'
    11. 'Användardefinierat fält 2'
    12. 'Användardefinierat fält 3'
  • Maximal längd på filnamn: Maximal längd för filnamn i listan över bilagor. Längre filnamn kommer att trunkeras i listan (enbart i visningen, de verkliga filnamnen ändras inte). Värdet 0 betyder ingen begränsning (men fältet för filnamn i databasen är begränsat till 80 tecken). Obs: Om filnamnen begränsas med denna inställning kommer det förkortade namnet att skrivas i fältet 'visat filnamn'.
  • CSS-stil för bilagornas tabellutseende: Ange ditt eget stilnamn här om vill använda en annan stil än den förvalda som är 'attachmentsList'. Se stycket [Design av bilagelistan] för mer information.
  • Sätt att öppna länkar till filer: Denna inställning styr hur länkade bilagor kommer att öppnas. 'I samma fönster' betyder att filen öppnas i samma webbläsarfönster. 'I nytt fönster' betyder att filen öppnas i ett nytt fönster. I vissa webbläsare betyder detta att filen öppnas under en ny flik.
  • Prefix till filnamn: Denna inställning styr om ett numeriskt prefix ska tillfogas filnamnet när filen lagras på servern. Det finns tre val:
    1. 'Artikel-ID' - Detta val gör att filen får artikelns ID som prefix. Det minskar risken för kolliderande filnamn och säkerställer att en viss artikels bilagor sorteras tillsammans i en kalaloglistning. Notera också att valet gör det möjligt att bifoga olika filer med samma namn till olika artiklar, men inte till samma artikel.
    2. 'Bilagans ID' - Detta val ger bilagans ID-nummer som prefix till filnamnet. Det förhindrar kolliderande filnamn eftersom varje bilaga alltid har ett unikt ID-nummer. Bilagorna kommer att sorteras i den ordning de har lagts till i kataloglistan.
    3. 'Inget' - Detta val skapar inget prefix och bilagan lagras med oförändrat filnamn. Det gör att risken ökar för kolliderande filnamn eftersom varje uppladdad fil måste ha ett unikt namn.
  • Underkatalog för uppladdning: Programmet 'Attachments' placerar sina filer i en underkatalog på roten till din Joomla! webbplats. Förvalt namn på katalogen är 'attachments'. Notera att enbart kommade uppladdningar påverkas on du byter till en annan katalog. Tidigare uppladdade filer ligger kvar i sin gamla katalog, och deras poster i databasen pekar dit. Om du behöver flytta alla filer till en annan katalog måste du göra det med en filhanterare och därefter manuellt uppdatera alla poster i databasen till att peka på den nya platsen.
  • Egna rubriker på listorna över bilagor: Som förvalt värde sätter programmet rubriken 'Bilagor' på listan över bilagor till en artikel (om sådana finns). I vissa fall kan en annan rubrik vara lämpligare. Du har möjlighet att ange en annan rubrik för varje enskild artikel. Om du till exempel vill ha rubriken 'Nedladdningsbara filer' i artikel 211, anger du här '211 Nedladdningsbara filer' (utan apostrofer). Skriv en post per rad.
     
    Anm.: Om du globalt vill ändra rubriken för alla bilagelistor kan du redigera din lokala språkfil nedan, i raden för 'ATTACHMENTS TITLE'. 
        administrator/language/qq-QQ/qq-QQ.plg_frontend_attachments.ini
    Där qq-QQ står för din gällande språkkod, exmpelvis sv-SE för svenska. (Om du inte är bekant md Joomla! språkfiler, hitta raden som har 'ATTACHMENTS TITLE' på vänster sida om tecknet '='. Ändra som du vill efter likhetsecket, men ändra inget före det.)
  • Göm bilagelistan för: Detta är en kommaseparerad lista på nyckelord eller Sektioner/Kategorier för vilka listan över bilagor inte kommer att visas. Två speciella nyckelord kan användas: 'frontpage' för att dölja bilagelistor på förstasidan, och 'all_but_article_views' för att visa listorna bara i artikelvyer. (Nyckelorden skrivs utan apostrofer). ID-nummer för Sektion/Kategori skrivs som siffror med ett snedstreck (/) emellan: Sektion#/Kategori#, Sektion#/Kategori#. Ange bara 'Sektion#' för att täcka alla dess Kategorier. Till exempel: 23/10, 23/11, 24
  • Säkra nedladdningsbara bilagor: I förvalt läge sparas bilagefilerna i en allmänt åtkomlig underkatalog. Om du väljer säker här så kommer underkatalogen att bli blockerad mot direkt åtkomst. Nedladdningslänkarna i användargränssnittet kommer att ladda ned filerna men blir indirekta länkar. Detta förhindar access för användare utan rätt behörighet. Om du inte väljer säkert läge visas länkarna enligt inställningarna ovan, men filerna kommer att bli tillgängliga för alla som känner till deras URL, eftersom underkatalogen är publik. Läget säker förhindrar access för icke behöriga användare, även om de vet filernas URL, eftersom valet förhindrar publik access till bilagornas underkatalog.
  • Nedladdningsmetod vid säker nedladdning: Detta val bestämmer om bilagorna kommer att laddas ned som separata filer, eller öppnas i webbläsaren (om webbläsaren kan hantera filtypen) Det finns två alternativ:
    • 'inline': I detta läge visas bilagor i webbläsaren, om webbläsaren kan visa filtypen (som till exempel text och bilder).
    • 'bilaga' : I detta läge kommer bilagorna alltid att laddas ned som separata filer.

    Filer som inte kan visas i webbläsaren kommer alltid att laddas ned som externa filer.

Visat Filnamn

Det normala är att visa hela filnamnet som en länk för att ladda ned registrerade bilagor. I vissa fall kan filnamnet vara för långt för att detta ska vara praktiskt. I formuläret för att ladda upp filer finns ett fält, 'Visat filnamn' där den som laddar upp filen kan ange ett alternativt namn eller benämning att visa istället för det fulla filnamnet. Här kan till exempel anges någon förkortning av filnamnet. Detta fält kan redigeras i administratörsgränssnittet för att redigera bilagor. Anm.: Det finns en annan inställning, "Maximal längd på filnamn" i instickprogrammets inställningar. Detta kan ställas in att automatiskt trunkera det visade filnamnet till en given längd; det trunkerade namnet kommer att skrivas till fältet 'Visat filnamn'.

Att formatera bilagelistorna med CSS

Listorna på bilagor i användargränssnittet görs med en speciell 'div'-klass som innehåller en tabell. Denna tabell är associerad med flera olika CSS-klasser för att ge webbutvecklaren full flexibilitet att formatera tabellen. Se exempel på hur det görs i insticksprogrammets CSS-fil (i plugins/content/attachments.css). Om du vill ändra på utseendet, gör gärna så att du kopierar originalstilarna till filens slut, och döp om 'attachmentsList' i den kopierade delen till något annat. Redigera parametern för Attachments (i komponenthanteraren) och ändra parametern attachments table style till ditt nya klassnamn. Ändra sedan definitionerna i den kopierade delen på det sätt du vill. Genom att göra så här kan du enkelt återfå originalutseendet genom att ändra parametern attachments table style till det tidigare förvalda 'attachmentList'. En annan fördel är att sektioenen med din ändrade stil kan kopieras till en fil och lätt hämtas tillbaka vid en versionsuppgradering av programmet. Detta kan också göras med ett CSS @import kommando. Ikoner för filtyper

Tillägget 'Attachments' visar en ikon för filtypen vid varje post i listan över bilagor. Gör så här om du vill lägga till en ikon för ytterligare filtyper: (1) Placera en lämplig ikon i katalogen 'media/attachments/icons', om där inte redan finns en för din önskade filtyp; (2) Redigera filen 'components/com_attachments/file_types.php' och lägg till en rad för din filtyp i listan $attachments_icon_from_file_extension som kopplar filtillägget till ett ikonnamn (från katalogen media/attachments/icons). Om det inte fungerar måste du kanske även lägga till en rad i $attachments_icon_from_mime_type. (3) Glöm inte att kopiera ikonfilen och din ändrade file_types.php till en extern katalog om du i framtiden uppgraderar Attachments. [Till sidans topp]

Varningar

  • Om du laddar upp bilagor med känslig eller privat information, se till att aktivera Säkra nedladdningsbara bilagor!
    Om du inte använder den funktionen lagras dina bilagor i en publik underkatalog och blir åtkomliga för var och en som vet den fulla URL-adressen. Valet säker hindrar åtkomst för alla som inte har rätt behörighet (enligt inställningarna ovan). Se avsnittet Säkra nedladdningsbara bilagor ovan för mer information.
  • Varje gång en fil laddas upp kontrolleras att lagringskatalogen existerar, och den skapas om den inte finns. Det förvalda värdet är 'attachments' i rotkatalogen för din webbplats. Namnet på underkatalogen kan ändras i 'Underkatalog för uppladdning'. Om tillägget 'Attachments' inte kan skapa sin underkatalog själv måste du göra det manuellt (och du kan få problem med att ladda upp filer). Se till att sätta lämpliga skrivrättigheter på katalogen i så fall. I Unix/Linux-världen innebär det troligen något i stil med 775. Observera att toppnivån på din webbplats kan ha begränsningar så att webservern (och PHP) inte tillåts att skapa underkataloger. Du kan behöva lätta på sådana restriktioner tillfälligt för att tillåta programmet att skapa sin underkatalog.
  • Om detta tillägg inte tillåter dig att ladda upp vissa filtyper (till exempel zip-filer) kan detta bero på att Attachments respekterar Mediahanterarens inställningar vilka filtyper som tillåts för uppladdning. Syftet är att stoppa potentiellt farliga filer som html eller php. Administratören kan ändra dessa inställningar under Globala inställningar i menyn för Webbplats, fliken System. Under Mediainställningar här kan önskade mime-typer läggas till i listorna.
  • Det andra insticksprogrammet, 'add_attachment_btn_plugin' (som kallas 'Editor Button - AddAttachment' i hanteraren för insticksprogram) gör att bilagor kan läggas till artiklar under redigering i redigeringsgränssnittet. Insticksprogrammet lägger till en knapp 'Lägg till bilaga' under redigeringsfönstret. Om du inte ser denna knapp fast du har installerat insticksprogrammet kan det bero på att det inte är aktiverat i menyn för Insticksprogram. Lägg märke till att knappen syns även för en helt ny artikel under redigering, men inte kan användas och visar en varning tills artikeln är sparad en gång. Artiklarna måste existera (ha fått ett ID) innan de kan få bilagor.
  • Om du inte kan se Attachments i användargränssnittet kan det ha flera skäl:
    • Insticksprogrammet 'Content - Attachments' är inte aktiverat. Gör det i hanteraren för Insticksprogram.
    • Inställningen för vilka som kan se bilagor är begränsad till inloggade användare, och du är inte inloggad. Inställningen ändras i Komponenthanteraren.
    • Insticksprogrammet 'Content - Attachments' har inte åtkomstnivån Publik (i hanteraren för Insticksprogram).
  • Om du får problem med storleksbegränsningar för filer du försöker ladda upp, prova att lägga till följande rader i filen .htaccess i rotkatalogen på din Joomla! webbplats: 
      php_value upload_max_filesize 32M
  php_value post_max_size 32M
    där du ändrar måttet 32M (megabytes) till vad du vill sätta som maximal filstorlek för uppladdning.
  • Det finns ett hjälpforum och ett FAQ-forum för Attachments på webbplatsen Joomlacode (på engelska). Rådfråga dessa om du stöter på något problem som inte behandlas i denna hjälpfil.
    • Attachments Forums at Joomlacode.org

Uppgradering

Följ denna anvisning om du vill uppgradera programmet och bevara redan registrerade bilagor:
  1. [Detta steg är valfritt men högst rekommendabelt, för att se till att du har en säkerhetskopia av databasen om något går fel.]
    Använd phpMyAdmin (eller annat verktyg för SQL redigering) för att spara innehållet i tabellen jos_attachments table (Använd valet 'Export' med 'Kompletta infogningar' för data (inte 'Utökade infogningar'). Du bör också säkerhetskopiera uppladdade bilagor (vanligen i underkatalogen 'attachments').
  2. Gå till hanteraren för Attachments i administratörsgränssnittet och klicka på knappen Admin längst till höger bland verktygen. Klicka på valet Avaktivera avinstallation av Attachments MySQL-tabell. Detta bevarar databasen vid en avinstallation av komponenten Attachments.
  3. Avinstallera komponenten 'Attachments'.
  4. Avinstallera alla insticksprogram som hör till 'Attachments'.
  5. Installera den nya programversionen genom att packa upp releasefilen och installera komponenten och alla insticksprogram. Databasen med bilagor kommer automatiskt att uppdateras under installationen.
  6. Aktivera alla 'Attachments' insticksprogram.
  7. Nu ska alla dina bilagor vara tillgängliga igen. Gå till hanteraren för Attachments i administratörsgränsnsnittet för att säkerställa att allt fungerar som det ska.

Lägg märke till att en avinstallation av tillägget 'Attachments' inte raderar tidigare uppladdade filer, men raderar programmets tabell i databasen. Så efter en ominstallation till en ny version av 'Attachments' är det bara nödvändigt att återställa den tabellen för att återställa alla bilagor.

Om du väljer att inte återställa databasen är det viktigt att du manuellt raderar överblivna uppladdade för att undvika framtida konflikter med filnamn i katalogen med bilagor. [Till sidans topp]
